# Judex – TS video stream (HLS) backend + frontend

Backend serves HLS (`.m3u8` + `.ts` segments); frontend plays the stream with hls.js. Data can be sourced via SSH or any other backend pipeline.

## Quick start

### 1. Backend

```bash
cd backend
npm install
```

Generate a sample HLS stream (requires [ffmpeg](https://ffmpeg.org/)):

```bash
chmod +x scripts/generate-sample-stream.sh
./scripts/generate-sample-stream.sh
# Or with your own file: ./scripts/generate-sample-stream.sh /path/to/video.mp4
```

Start the server:

```bash
npm run dev
```

Backend runs at **http://localhost:3001**. HLS playlist: **http://localhost:3001/stream/playlist.m3u8**.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at **http://localhost:5173** and is proxied to the backend for `/api` and `/stream`, so the default stream URL works without CORS.

### 3. Play

Open http://localhost:5173. Use the default URL or paste `http://localhost:3001/stream/playlist.m3u8` and click **Load**.

## Project layout

| Path | Purpose |
|------|--------|
| `backend/` | Express server: serves HLS from `stream-output/`, CORS, optional SSH/source integration |
| `backend/stream-output/` | HLS files: `playlist.m3u8` and `segment*.ts` (generated by script or your pipeline) |
| `frontend/` | React + Vite + hls.js player; configurable stream URL |

## Sourcing the stream (e.g. via SSH)

The backend does **not** implement SSH itself. You can:

1. **Pre-generate HLS**  
   Run `scripts/generate-sample-stream.sh` or your own ffmpeg/encoder that writes into `backend/stream-output/`.

2. **Proxy from another service**  
   Add an endpoint that fetches `.m3u8` / `.ts` from a service that itself gets the TS over SSH (e.g. a separate process that SSHs and runs ffmpeg, then serves or writes segments).

3. **Live pipeline**  
   Run a process that receives TS over SSH, segments it into HLS (e.g. with ffmpeg), and writes to `stream-output/` while the backend serves that directory.

Environment:

- **Backend**: `PORT`, `PUBLIC_URL` (for `GET /api/stream-url`).
- **Frontend**: `VITE_STREAM_URL` to override the default stream URL (e.g. for production).

## Scripts

- **Backend**: `npm start` (run), `npm run dev` (run with watch).
- **Frontend**: `npm run dev`, `npm run build`, `npm run preview`.
